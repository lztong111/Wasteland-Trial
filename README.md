# 荒原试炼

荒原试炼是一个基于 React、TypeScript、Babylon.js 与 Havok Physics 的第三人称动作 RPG 原型，当前重点验证“移动、战斗、波次、成长、存档”组成的最小可玩闭环。

## 快速开始

环境要求：Node.js 20.19 及以上（或 Node.js 22.12 及以上）、npm 10 及以上，并使用支持 WebGL 2 与 WebAssembly 的现代浏览器。

```bash
npm install
npm run dev
```

打开终端输出的本地地址即可进入游戏。首次进入会显示玩法说明，点击“开始试炼”后才会接收键盘和鼠标输入。

## 当前玩法

1. 击败第一波 3 名守卫。
2. 靠近补给箱按 `E`，获得生命药水并召来两名重装守卫和一名小首领。
3. 清除第二波后靠近神龛按 `E`，恢复生命并完成本轮试炼。
4. 在结算界面三选一：生命上限、基础伤害或体力上限。选择后才能进入下一轮。
5. 击杀守卫获得经验，升级会恢复生命并提高生命上限和基础伤害；后续轮次会提高敌人的生命、伤害和经验奖励。

近战攻击会按输入节奏组成三段连击，重击消耗更多体力，远程攻击会朝准星方向发射投射物。敌人攻击前会出现明显前摇，离开攻击范围可以取消本次攻击；闪避期间拥有短暂无敌时间。

普通守卫用于第一波教学，重装守卫生命更高但移动更慢，小首领会在第二波最后出现并必定掉落遗迹重刃；重装守卫有概率掉落守卫裂刃。

## 操作

| 行为 | 键盘或鼠标 |
| --- | --- |
| 相机相对移动 | `W` / `A` / `S` / `D` |
| 跳跃 | 空格 |
| 方向闪避 | `Shift` |
| 三段轻击 | 鼠标左键 |
| 蓄力重击 | `Q` |
| 远程攻击 | 鼠标右键 |
| 交互、开启箱子或神龛 | `E` |
| 打开背包 | `I` 或 `Tab` |
| 展开操作说明 | `H` |
| 暂停或退出鼠标锁定 | `Esc` |

移动端会自动显示虚拟方向键和战斗按钮。UI 面板会阻止事件穿透到游戏画布，避免点击背包或设置时误触攻击。

## 技术结构

```text
src/config     集中管理玩法和渲染数值
src/core       Babylon 引擎、物理、相机和游戏循环
src/entities   玩家实体和物理胶囊
src/models     程序化模型、武器与 GLB 动画管线
src/systems    输入、战斗、敌人、交互和音频
src/data       RPG 状态、关卡进度、设置与存档
src/ui         React HUD、玩法说明、暂停和结算界面
src/world      场景灯光、静态物件与碰撞体
```

React 只负责 UI，Babylon 场景和战斗逻辑独立运行。Havok 与 GLB loader 均采用动态导入，默认首屏不会加载未使用的模型解析代码。配置外部角色模型时，设置 `src/config/gameConfig.ts` 中的 `assets.playerModel`，loader 会按 `Idle`、`Run`、`Attack`、`Hit`、`Death`、`Dodge` 及常见别名匹配动画；模型缺失或格式不兼容时会自动回退到程序化角色。

## 存档与配置

存档使用浏览器 `localStorage`，当前键名和版本由 `gameConfig.save` 统一管理。存档包含 RPG 属性、背包、装备、波次、箱子、神龛、结算和升级选择，并在数据恢复前校验数值边界、物品结构和关卡顺序。

所有可调玩法数值集中在 `src/config/gameConfig.ts`，包括敌人生命成长、攻击前摇、体力消耗、波次目标、掉落和渲染缩放。修改配置后无需改动系统实现。

## 后续路线

当前执行顺序为 `P0 → P2 → P1`。P0 关注物理、事件释放、参数边界和核心单测；P2 已完成按需资源、移动端、GLB、存档和端到端验证；P1 已加入敌人类型、掉落装备和波次小首领，后续可继续扩展更多远程敌人、套装和关卡事件。
